# Landscape

Landscape的本质是一个巨大的连续网格（Mesh）。由于你不能像对待普通静态网格体那样，简单地给不同的多边形分配不同的材质（这会导致硬边缘和巨大的渲染开销），虚幻采用了**纹理泼溅（Texture Splatting）** 或称为**权重图混合（Weightmap Blending）** 的技术。
## 1. 为什么一个地形材质中要包含那么多“子材质”？

因为整个地形（或者说地形的每个 Component 组件）在渲染时，**只能应用一个材质实例**。

为了让地形上同时出现草地、泥土、岩石和雪，你必须在一个超级材质（Landscape Material）中把它们全部定义好。在这个材质中，我们通常会使用 `Landscape Layer Blend`（地形层混合）节点。

这个节点的作用是相当于一个**多路复用器（Multiplexer）**。它接收多种贴图（如草地反照率、泥土反照率），然后根据地形系统提供的**权重图（Weightmap）**（本质上是黑白遮罩贴图），在像素级别计算出当前位置应该显示什么地貌。

## 2. Weight Blend 与 No Weight Blend

在配置 `Landscape Layer Blend` 节点时，每添加一个层（Layer），你需要指定它的混合模式。这决定了当你用画刷在地形上涂抹时，引擎如何处理不同层之间的权重冲突：

* **Weight Blend (权重混合 / 标准混合)：**
* **逻辑：** 所有设置为 Weight Blend 的层，它们在任意一点的权重**总和必须等于 1（100%）**。
* **表现：** 如果你在已经铺满“草地（100%）”的地方画上“泥土（50%）”，引擎会自动把该处的“草地”权重降低到 50%。这是一种**排他性**的混合，非常适合基础地貌的过渡（泥土、草地、沙子）。


* **No Weight Blend (Alpha混合 / 非权重混合)：**
* **逻辑：** 这种层的权重是**独立**的，不参与上述的“总和等于1”的计算。
* **表现：** 当你画这种层时，它会像贴纸一样直接覆盖在现有的地貌上，而不会去强行扣减底部地貌的权重。它非常适合**细节覆盖物**，比如地上的水坑（Puddles）、积雪（Snow）或者落叶。不管下面是泥土还是石头，这层雪直接叠加在最上面。

## 3. LayerInfo 文件

当你在材质中设置好了这些 Layer 后，材质本身只是一套“渲染规则”。但在你实际去地形上“画”草地或泥土时，引擎需要地方来存储你的**绘制数据（画刷权重信息）**。

这就引出了 **LayerInfo（地形层信息对象）**。

* **LayerInfo 是什么？** 它是独立的数据资产（Data Asset）。它告诉引擎：“这个叫‘Grass’的层，对应的物理材质（Physical Material，用于区分脚步声或物理摩擦）是什么？它的混合行为是什么？”
2.  **资产复用：** 如果你在同一个项目里创建了多个不同的地形（比如多个关卡），它们共用同一个地形材质时，就可以直接复用 `_sharedassets` 里的 LayerInfo，而不需要每次都重新定义物理材质和层级行为。

# 命名规范

## PascalCase格式

驼峰命名法：

| 原始单词                         | PascalCase (帕斯卡)             | camelCase (小驼峰)              | snake_case (蛇形)    |
| ---------------------------- | ---------------------------- | ---------------------------- | ------------------ |
| `player` + `controller`      | **P**layer**C**ontroller     | **p**layer**C**ontroller     | player_controller  |
| `get` + `actor` + `location` | **G**et**A**ctor**L**ocation | **g**et**A**ctor**L**ocation | get_actor_location |
| `ip` + `address`             | **I**p**A**ddress            | **i**p**A**ddress            | ip_address         |

## 首字母

多数情况下，UHT需要正确的前缀，因此添加正确的前缀至关重要；

类型名前缀使用额外的大写字母，用于区分类型与变量，例如`FSkin`为类型名，`Skin`则为实例名；

| 说明           | 前缀  |
| ------------ | --- |
| 模板类          | T   |
| 继承自UObject的类 | U   |
| 继承自AActor的类  | A   |
| 继承自SWidget的类 | S   |
| 抽象界面类        | I   |
| 概念类结构类型      | C   |
| 枚举           | E   |
| 布尔值          | b   |
| 传入参数         | In  |
| 传出参数         | Out |

c++11中智能指针的自定义实现，旨在减轻内存分配和追踪的负担；`UObject`使用更适合游戏代码的单独内存追踪系统，因此**这些智能指针无法与`UObject`系统同时使用**；

# 指针类型

## 共享指针 TSharedPtr

共享指针拥有其引用的对象，防止该对象被删除，并在无共享指针或共享引用时删除该对象；
共享指针可为空白，意味其不引用任何对象；
任何非空共享指针都对其引用的对象生成共享引用；

## 共享引用 TSharedRef

共享引用的行为与共享指针类似，拥有其自身引用的对象；
共享引用不能引用空对象；

## 弱指针 TWeakPtr

弱指针类似共享指针，但不拥有其引用的对象，因此不影响对象的生命周期；

## 唯一指针 TUniquePtr

唯一指针会显式拥有其引用的对象，同时只能有一个唯一指针指向给定对象，因此唯一指针只能转移所有权，不能共享；

# 助手类与函数

## TSharedFromThis

*让对象能安全地引用自己*

当一个类被 `TSharedPtr` 管理时，如果这个类的成员函数想要把 “我自己（`this`）” 传递给其他函数（比如绑定委托、要把自己加入到一个数组里），你不能直接传 `this` 指针。
为什么？因为 `this` 是裸指针，接收方无法知道这个对象是被智能指针管理的，这可能导致引用计数错误或悬空指针。

`TSharedFromThis` 允许你在类内部安全地获取指向自己的 `TSharedRef` 或 `TSharedPtr`；

```cpp
class FMyWorker : public TSharedFromThis<FMyWorker> // 1. 继承
{
public:
    void DoWork()
    {
        // 假设有一个全局管理器需要注册工人
        // Manager->Register(this); // 错误！这是裸指针，不安全
        
        // 正确：创建一个指向自己的共享引用
        TSharedRef<FMyWorker> SelfRef = AsShared(); 
        
        UE_LOG(LogTemp, Log, TEXT("我正在把我自己(智能指针)传递出去"));
    }
};

// --- 使用场景 ---
void TestSharedFromThis()
{
    // 关键点：对象必须先被智能指针拥有！
    TSharedPtr<FMyWorker> Worker = MakeShared<FMyWorker>();
    
    Worker->DoWork(); // 这里面调用 AsShared() 是安全的
    
    // 错误示范：
    // FMyWorker* RawWorker = new FMyWorker();
    // RawWorker->DoWork(); // 崩溃！因为 RawWorker 没有被智能指针管理，AsShared() 找不到引用计数器。
}
```

## MakeShared/MakeShareable

*用于创建TSharedPtr或TSharedRef指针*

- `MakeShared`：推荐，性能更好；
	原理： 在一次内存分配中同时分配 **对象本身** 和 **引用计数控制器**；
	优点： 效率高，缓存亲和性好（Cache Friendly），减少内存碎片；
	限制： 只能用于构造函数是 public 的类（或者声明了友元）；

- `MakeShareable`：兼容性更好；
	原理： 它接受一个现有的裸指针（通常是 new 出来的）。这意味着**发生了两次内存分配**（一次是 new 对象，一次是分配引用计数器）；
	场景： 
	1.  构造函数是 private 的（无法用 MakeShared）；
	2.  你需要接管一个第三方库创建的裸指针；
	3.  你需要自定义删除器（Deleter）；


```cpp
class FMyData
{
public:
    FMyData(int32 InVal) : Value(InVal) {}
    int32 Value;
};

void TestCreation()
{
    // 方式 A: MakeShared (首选)
    // 就像 std::make_shared，直接把构造参数传进去
    TSharedPtr<FMyData> Ptr1 = MakeShared<FMyData>(100);

    // 方式 B: MakeShareable (次选)
    // 先 new，再包起来。
    FMyData* RawPtr = new FMyData(200);
    TSharedPtr<FMyData> Ptr2 = MakeShareable(RawPtr);
    
    // 特殊用法：自定义删除器 (比如管理非内存资源)
    TSharedPtr<void> FileHandle = MakeShareable(OpenMyFile(), [](void* Ptr){
        CloseMyFile(Ptr); // 当智能指针销毁时，自动调用关闭文件
    });
}
```

## CastShared

`StaticCastSharedPtr / StaticCastSharedRef`
	作用： 对应 C++ 的 static_cast。通常用于 向下转型（Base -> Derived）。
	安全性： 不安全（无运行时检查）。你必须自己在逻辑上 100% 确定这个基类指针确实指向那个子类对象。如果转错了，行为未定义（但比 C++ 原生转换稍微安全一点点，因为引用计数还在）。
	注意：如果需要运行时类型检查（类似 dynamic_cast），请使用 DynamicCast（仅限 UObject）或自己在纯 C++ 类中实现 RTTI。对于纯 C++ 智能指针，通常不做 DynamicCast，而是依靠设计保证。

`ConstCastSharedPtr / ConstCastSharedRef`
	作用： 对应 C++ 的 const_cast。用于移除 const 属性。
	场景： 当你手里有一个 `const TSharedPtr<const FMyClass>`，但你需要调用它的非 const 函数时；

```cpp
class FBase { virtual ~FBase() {} };
class FDerived : public FBase { public: void SayHello() {} };

void TestCasting()
{
    // --- 1. Static Cast (向下转型) ---
    TSharedPtr<FBase> BasePtr = MakeShared<FDerived>(); // 实际存的是子类

    // 这里的转换是基于我们知道它肯定是 FDerived
    TSharedPtr<FDerived> DerivedPtr = StaticCastSharedPtr<FDerived>(BasePtr);
    
    if (DerivedPtr.IsValid())
    {
        DerivedPtr->SayHello();
    }

    // Ref 版本 (如果 BasePtr 无效会崩溃，更严格)
    TSharedRef<FBase> BaseRef = BasePtr.ToSharedRef();
    TSharedRef<FDerived> DerivedRef = StaticCastSharedRef<FDerived>(BaseRef);


    // --- 2. Const Cast (去常量化) ---
    TSharedPtr<const FDerived> ConstPtr = MakeShared<FDerived>();
    
    // ConstPtr->SayHello(); // 编译错误！SayHello 不是 const 函数
    
    // 强制移除 const
    TSharedPtr<FDerived> MutablePtr = ConstCastSharedPtr<FDerived>(ConstPtr);
    MutablePtr->SayHello(); // 现在可以调用了
}
```


# 线程安全

通常仅在单线程上访问智能指针的操作才是安全的。如需访问多线程，请使用智能指针类的线程安全版本：
    `TSharedPtr<T, ESPMode::ThreadSafe>`
    `TSharedRef<T, ESPMode::ThreadSafe>`
    `TWeakPtr<T, ESPMode::ThreadSafe>`
    `TSharedFromThis<T, ESPMode::ThreadSafe>`

由于原子引用计数，此类线程安全版本比默认版本稍慢，但其行为与常规C++指针一致：
    读取和复制固定为线程安全；
    写入和重置须同步后才安全；

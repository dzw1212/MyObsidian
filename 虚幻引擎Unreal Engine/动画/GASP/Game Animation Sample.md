`GASP` 是 Epic Games为展示其最新的、最先进的动画技术而发布的一个**官方示例项目**。它的核心目的在于提供一个**开箱即用、高质量、高性能**的角色运动系统，并作为开发者学习和集成这些新技术的**起点或参考框架**。


`GASP` 不仅仅是一个简单的角色移动蓝图，它集成了虚幻引擎在角色动画领域最前沿的几个技术，其中最主要的是：

*   **Motion Matching (运动匹配)**：
    *   GASP 是 Epic 官方重点展示如何使用其内置 **Pose Search** (姿态搜索) 和 **Motion Matching** 功能的样本。它不再依赖传统的庞大状态机和混合空间，而是通过实时搜索海量动画数据中与当前角色状态和意图最匹配的姿态来驱动动画，从而实现**极度自然和流畅**的动作过渡，这正是它与 ALS Refactored 这种传统混合空间框架最大的区别。
*   **Layering System (分层系统)**：
    *   GASP 采用了高效的分层系统，可以将行走/奔跑等**底层动作**（Locomotion）与瞄准、持枪、挥拳等**上层动作**（Overlay/Additive Animation）独立地混合和叠加，提高了动作的复杂性和模块化程度。
*   **Procedural Tools (程序化工具)**：
    *   GASP 中也包含了用于修正动画缺陷的程序化工具，例如**Foot IK（足部反向运动学）**，用于确保角色在不平坦地面行走时，脚部能准确地踩在地面上，减少滑步。

对开发者的价值：

*   **AAA 级运动系统起点**：它提供了一个包含数百个高质量动画（例如 5.5 版本包含 500+ 个 AAA 质量动画）的完整角色控制器。开发者可以直接将 GASP 作为一个基础框架，在其上构建自己的游戏逻辑和特定动画，而无需从头编写复杂的运动系统。
*   **学习和参考**：开发者可以深入研究 GASP 的动画蓝图（AnimGraph）、姿态搜索数据库（Pose Search Databases, PSD）和姿态搜索架构（Pose Search Schemas, PSS），学习如何在高保真度的 AAA 级项目中组织和驱动复杂的角色动画。
*   **性能优化**：虽然 Motion Matching 需要大量数据，但 GASP 的设计目标之一是展示如何以相对较小的 Motion Matching 数据库和混合栈（Blend Stack）实现高性能的运动，例如在某些配置下可以切换到基于 State Machine 的方法来减少计算负担。

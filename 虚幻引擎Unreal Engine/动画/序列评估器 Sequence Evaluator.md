
如果说 `Sequence Player`（序列播放器）是一台**自动播放**的电影放映机，那么 `Sequence Evaluator` 就是一个可以**手动拨动胶卷**的剪辑台。

`Sequence Evaluator` 的唯一目标是：**根据你输入的具体时间（Explicit Time），输出该时间点对应的动画姿态（Pose）。**

它**不会**随着时间流逝自动播放动画。如果你给它的 `Explicit Time` 输入 0.5 秒，它就永远停在 0.5 秒的那一帧，直到你把输入值改为其他数字。


# 对比Sequence Player

| 特性        | Sequence Player (播放器)           | Sequence Evaluator (评估器)            |
| :-------- | :------------------------------ | :---------------------------------- |
| **驱动方式**  | **时间驱动**：随游戏时间（Delta Time）自动增加。 | **数据驱动**：随输入的 `Explicit Time` 变量改变。 |
| **逻辑复杂度** | 简单：连上就能动。                       | 较高：需要你自己写逻辑计算“当前该播到几秒”。             |
| **循环处理**  | 自动循环。                           | 无循环概念（除非你手动把时间重置回 0）。               |
| **根骨骼位移** | 自动累加位移。                         | 需配合 `Teleport` 属性手动控制位移增量。          |
| **性能**    | 略高（有内部状态更新）。                    | **极高**（无状态更新，只是简单的查找采样）。            |

# 核心应用场景

它是实现**高级动画系统**（如运动匹配、物理驱动动画）的基石。

## 距离匹配
这是最常见的用法。通过 `Advance Time by Distance Matching` 计算出的时间传给 `Sequence Evaluator`。
*   **逻辑：** 物理移动了 1 米 -> 计算出动画应在 0.3 秒 -> 传给 Evaluator。
## 手动驱动的 UI 动画或交互
比如一个开门动画，你想让门的开启程度完全跟随玩家鼠标滚轮的滑动：
*   **逻辑：** 鼠标滚动值 -> 映射到动画时间 -> 传给 Evaluator。

## 复杂的起步/停止逻辑
在高质量 Locomotion 系统中，起步动画往往不是线性播放的。开发者会根据脚部与地面的接触情况，动态地“拉伸”或“压缩”动画时间，这时必须用 Evaluator。

# 关键属性
# Teleport to Explicit Time

 **`Teleport to Explicit Time`**（瞬间移动至显式时间）属性的作用，主要决定了**当你改变显式时间时，引擎如何处理根骨骼位移（Root Motion）和动画曲线的增量。**

*   **如果不勾选（False - 默认）：**
    引擎会认为你是“平滑播放”或“擦除（Scrubbing）”到这个时间的。
    *   **根骨骼位移：** 如果上一帧时间是 0.1s，这一帧你设为 0.2s，引擎会计算这两帧之间的位移差（Delta），并将其应用到角色身上（让角色在世界空间移动）。
    *   **动画通知：** 可能会触发 0.1s 到 0.2s 之间的所有动画通知（Notifies）。

*   **如果勾选（True）：**
    引擎会认为你是直接“闪现”到这个时间的，不经过中间过程。
    *   **根骨骼位移：** **不会产生位移增量。** 即便你这一帧从 0.1s 直接跳到了 0.5s，角色在场景中的位置也不会移动，它只是简单地把姿态（Pose）更新到了 0.5s 的样子。
    *   **动画通知：** 通常会跳过中间路径上的通知，直接跳到目标点。

---
在距离匹配中：

*   **场景：** 假设你的角色刚刚结束了一段跑步，`Explicit Time` 停在 0.8s。现在你想让它重新开始一段新的起步动画，并将 `Explicit Time` 重置为 0。
*   **如果不勾选 Teleport：** 引擎会计算从 0.8s 回到 0.0s 的“负向位移”。这会导致你的角色在这一帧产生一个剧烈的反向瞬间抖动（因为根骨骼位移被倒着计算了）。
*   **如果勾选 Teleport：** 当你把时间设回 0 时，勾选此项，角色会安静地回到起始姿态，而不会在世界空间产生任何物理位移。

贴图动画在3D模型和角色渲染的语境下，通常指的是**顶点动画贴图（Vertex Animation Textures，简称 VAT）**。

VAT 是一种将模型的动画数据（每个顶点在每一帧的位置和法线信息）烘焙成一张或多张2D贴图，然后在引擎的**材质（Material）** 中读取这些像素颜色，并通过**世界位置偏移（World Position Offset, WPO）** 在 GPU 端还原动画的技术。

# 与 骨骼动画 的核心区别

这两种技术解决的是不同的渲染瓶颈。简单来说：**骨骼动画追求交互和细节，贴图动画追求极致的数量（性能）。**

| 特性 | 骨骼动画 (Skeletal Animation) | 贴图动画 (Vertex Animation Texture) |
| --- | --- | --- |
| **驱动方式** | CPU 计算骨骼变换，GPU 蒙皮 (Skinning) | 纯 GPU 材质计算 (World Position Offset) |
| **模型类型** | 骨骼网格体 (`Skeletal Mesh`) | 静态网格体 (`Static Mesh`) |
| **性能开销** | 较高。受限于骨骼数量和 Draw Call，同屏几百个就会遭遇性能瓶颈。 | 极低。配合实例化 (ISM/HISM) 或 Niagara，同屏可以渲染**数万甚至数十万**个动画单位。 |
| **动画控制** | 极强。支持状态机(AnimBP)、混合(Blending)、IK、物理碰撞。 | 极弱。只能通过材质参数控制播放速度、循环和时间偏移，**无法平滑混合动画或使用IK**。 |
| **内存/显存占用** | 占用内存记录动画序列曲线。 | 占用显存记录高精度贴图（贴图分辨率 = 顶点数 × 动画帧数）。 |
| **适用场景** | 玩家主角、主要 NPC、需要复杂交互的怪物。 | 远景人群、大规模军队(如《全面战争》)、飞鸟鱼群、流体爆炸特效、破碎特效。 |

# VAT的工作原理

理解 VAT 的关键在于：**将空间坐标转化为颜色值**。

* **X 坐标** -> 贴图的 **R (红) 通道**
* **Y 坐标** -> 贴图的 **G (绿) 通道**
* **Z 坐标** -> 贴图的 **B (蓝) 通道**

贴图的**宽度（X轴）** 通常代表模型的**顶点编号**，**高度（Y轴）** 代表动画的**时间帧**。材质在运行时，根据当前的时间（Time），采样对应那一行的像素颜色，并将其作为偏移量加到原本的顶点位置上，模型就“动”起来了。

# AnimToTexture插件

过去，使用 VAT 必须借助外部软件（如 Houdini 的 SideFX Labs 工具或 3ds Max/Blender 脚本）来烘焙。但在 **UE5** 中，Epic 官方提供了一个极其方便的内置插件：**AnimToTexture**。

以下是使用 UE5 内置 **AnimToTexture** 插件制作和使用贴图动画的简要流程：

## 第 1 步：开启插件

1. 在 UE5 菜单栏选择 `Edit` -> `Plugins`。
2. 搜索 **AnimToTexture** 并启用，重启引擎。

## 第 2 步：准备数据并生成 VAT

1. 准备好你的 `Skeletal Mesh`（骨骼模型）和一段 `Anim Sequence`（动画序列，比如跑步）。
2. 在内容浏览器右键，创建一个 **Data Asset**，类选择 `AnimToTextureBakeDataAsset`。
3. 打开这个 Data Asset，配置你要烘焙的数据：
	* **Skeletal Mesh**: 填入你的模型。
	* **Anim Sequences**: 添加你的动画序列。
	* **Options**: 选择模式（通常选 `Vertex` 即可，如果要保留骨骼层级可选 `Bone`）。
4. 点击界面上的 **Bake** 按钮。引擎会自动生成：
	* 一个转换好的 **Static Mesh**（包含特定的 UV 布局）。
	* 位置贴图（Position Texture）和法线贴图（Normal Texture）。
	* 一个预设好的材质实例（Material Instance）。

## 第 3 步：关键的贴图设置

VAT 对贴图的压缩设置要求极其严格，因为颜色代表的是绝对的物理坐标，任何压缩带来的色偏都会导致模型“爆炸”或扭曲。

* 打开生成的贴图，确保 **Compression Settings** 设置为 `VectorDisplacementmap` 或 `HDR`。
* 确保 **sRGB** 被**取消勾选**。
* 确保 **Filter**（过滤模式）设置为 `Nearest`（最近邻），不要使用线性过滤，否则顶点插值会错乱。
*(注：如果使用 AnimToTexture 插件，这些通常会自动设置好，但如果是从外部导入，必须手动检查。)*

## 第 4 步：在场景中使用

1. 将生成的 **Static Mesh** 拖入场景。
2. 赋予生成的材质实例。
3. 你可以在材质实例的参数中调整播放速度（Play Rate）、播放偏移（Time Offset）等。
4. **终极应用**：将其放入 **Niagara** 粒子系统中作为 `Mesh Renderer` 发射，利用粒子的随机参数（如随机的起始时间和播放速度）驱动材质，你就能瞬间得到成千上万个动作各异的人群！

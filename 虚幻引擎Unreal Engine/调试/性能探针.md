
# TRACE_CPUPROFILER_EVENT_SCOPE

这是虚幻引擎5时代最推荐的打点方式，用于配合 **Unreal Insights** 进行深度分析。

**1. 代码实现**
你只需要在想测量的函数或代码块开始处加上这个宏即可：

```cpp
void AMyActor::TestFunction()
{
    // 添加探针，名称将显示在Unreal Insights中
    TRACE_CPUPROFILER_EVENT_SCOPE(AMyActor::TestFunction);

    // 你的逻辑代码...
    for (int32 i = 0; i < 10000; i++)
    {
        // 一些耗时操作
    }
}
```
**支持嵌套**：你可以在一个函数内的多个代码块分别加探针，Insights中会清晰地显示层级关系和耗时。

**2. 数据捕获与分析**

- **启动捕获**：运行游戏后，在控制台输入 `Trace.Start` 或在Unreal Insights中点击“Start Trace”按钮开始记录。
- **执行操作**：在游戏中进行你想要分析的操作。
- **停止并分析**：在控制台输入 `Trace.Stop` 停止记录。
- **查看结果**：打开 `UnrealInsights.exe`（通常位于 `Engine/Binaries/Win64/`），它会自动加载生成的`.utrace`文件。在“Timing Insights”视图的搜索框中输入你的函数名，就能精确定位到它的耗时。

# SCOPE_CYCLE_COUNTER

这个方法在旧版本引擎中很常见，适合在游戏运行时通过 `stat` 命令实时查看数据。

**1. 代码实现**
你需要先声明一个静态的统计组和计数器。

- **步骤1：声明** (通常放在头文件或类的cpp文件顶部)
```cpp
// 声明一个名为 "TestTime" 的统计组
DECLARE_STATS_GROUP(TEXT("TestTime"), STATGROUP_TestTime, STATCAT_Advanced);
// 在 "TestTime" 组下声明一个名为 "TestMyActorTick" 的计数器
DECLARE_CYCLE_STAT(TEXT("TestMyActorTick"), STAT_TestMyActorTick, STATGROUP_TestTime);
```

- **步骤2：使用** (在Tick或任意函数中)
```cpp
void AMyActor::Tick(float DeltaTime)
{
    Super::Tick(DeltaTime);
    // 放置探针，测量从这里到函数结束的耗时
    SCOPE_CYCLE_COUNTER(STAT_TestMyActorTick);

    // 你的逻辑代码...
}
```

**2. 实时查看**
在游戏或编辑器视口（或控制台）中输入 `stat TestTime`，屏幕上会实时显示 `TestMyActorTick` 这一项的平均、最小和最大耗时。
